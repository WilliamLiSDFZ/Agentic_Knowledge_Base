# GOLD: Generalized Knowledge Distillation via Out-of-Distribution-Guided Language Data Generation

**Source**: https://aclanthology.org/2024.findings-naacl.272/

## [POSITIVE] Iterative OOD-Guided Feedback
An iterative mechanism that identifies failure modes (OOD samples) of the small language model (SLM) and provides feedback to the large language model (LLM) to generate more diverse training data in subsequent iterations.

**Delta**: +4% average accuracy over ZeroGen and ProGen on classification tasks; +14% over LLM few-shot
**Condition**: Applicable to any NLP task (classification and sequence-to-sequence) where LLM generates training data for an SLM.

**Evidence**: "Our method (with an average accuracy of 67.1%) improves the performance of the pre-trained SLM... Our method demonstrates a 4% improvement over ZeroGen and ProGen, and also 10% improvement over P2Model."

## [POSITIVE] Energy-Based OOD Evaluation
Uses free energy scores from the SLM's output logits to detect OOD samples without requiring labels, making it robust to noisy labels generated by the LLM.

**Delta**: Outperforms baselines; robust to noisy labels
**Condition**: Used for selecting feedback samples from the validation set in each iteration.

**Evidence**: "The energy score does not require the labels of the generated sample and therefore is not prone to pick the data with noisy labels as the OOD samples."

## [POSITIVE] Symmetric Cross-Entropy (SCE) Loss
A noise-robust loss function combining cross-entropy and reverse cross-entropy to handle noisy labels in generated data.

**Delta**: +3% accuracy on RTE and MNLI compared to without SCE
**Condition**: Used for training the SLM on generated data with potential label noise.

**Evidence**: "Ablating the noise-robust SCE loss (i.e., V2) decreases the accuracy by about 3% on RTE and MNLI datasets."

## [POSITIVE] OOD Validation Set Generation
Generates a separate validation set that is significantly different from the training set in topic and style, using a prompt that asks for OOD samples.

**Delta**: Improves generalizability; outperforms baselines
**Condition**: Applied in each iteration to evaluate the SLM and identify failure modes.

**Evidence**: "The validation data at iteration t is generated by the prompt P_val = P_task ⊕ X_real ⊕ X_train[t] ⊕ P_ood as follows: ... where P_ood prompts the generation of new data that significantly diverges from X_train[t]."

## [POSITIVE] Task-Agnostic Framework
A framework that treats all tasks (including classification) as sequence-to-sequence, allowing application to any NLP task without modification.

**Delta**: Applicable to novel tasks like NL4OPT; outperforms prior arts
**Condition**: General design choice for all experiments.

**Evidence**: "Proposing a task-agnostic framework for data generation and KD from LLMs to SLMs that is applicable to any NLP task, even novel tasks."

## [NEUTRAL] Fixed General Prompt with 3 Real Samples
Uses a fixed prompt containing task definition and 3 real data samples for all methods, ensuring fair comparison.

**Delta**: No quantitative delta; ensures fair comparison
**Condition**: Applied consistently across all baselines and methods.

**Evidence**: "For all the experiments in this paper, we use a fixed, general prompt including a task definition and 3 samples of real data."

## [POSITIVE] Threshold-Based OOD Sample Selection (α and β)
Selects OOD samples with energy scores between lower (α) and upper (β) thresholds to exclude very noisy or high-likelihood samples.

**Delta**: Prevents drastically OOD samples; improves performance
**Condition**: Used in the feedback selection step; fixed thresholds (50%-80%) across all tasks.

**Evidence**: "We implemented an upper-bound threshold for selecting OOD samples... specifying the upper threshold as 50% guarantees that all of the excluded samples are high likelihood."

## [POSITIVE] Using LLaMA2-7B as LLM and T5-base as SLM
Default model choice: LLaMA2-7B as teacher and T5-base (220M) as student.

**Delta**: Outperforms prior arts; larger LLM (70B) further improves results
**Condition**: Default setting; ablation shows larger LLM (70B) yields better SLM performance.

**Evidence**: "We respectively use LLaMA2-7B and pre-trained T5-base (220M) as the LLM and SLM for all our experiments."

## [POSITIVE] Iterative Data Generation with 375 Iterations (3K samples)
Generates 3K samples over 375 iterations, with each iteration producing a batch of data.

**Delta**: Optimal at 3K samples; more samples do not always improve performance
**Condition**: Default setting; ablation shows diminishing returns beyond 3K for pre-trained tasks.

**Evidence**: "Unless otherwise mentioned, for the experiments... we generated 3K samples using the LLM in 375 iterations."

## [POSITIVE] Low Temperature (Default=1) for LLM Generation
Uses default temperature (1) for LLM data generation, unlike P2Model which uses high temperature for diversity.

**Delta**: Higher accuracy than high-temperature generation (80.3% vs 38.6% on QNLI)
**Condition**: Applied during LLM data generation to maintain accuracy.

**Evidence**: "For our method, where the LLM temperature is fixed to the default value of 1, we achieved an accuracy of 80.3% and 72.1% for the same tasks."
